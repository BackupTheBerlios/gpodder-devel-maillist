> (b) Not all podcasts got synced. Episodes with a yellow star seem to
> have been ignored. Any idea why?

If you mean this icon with the "yellow star", it means that this episode
is "new in the feed", i.e. you have to download it first, so it gets a
icon that looks like a file (music, audio, document - depending on which
file you downloaded):

http://img517.imageshack.us/img517/8016/iconstocknewli3.png

>From a usability point of view, do you think that the "yellow star" icon
is misleading and looks like the episode is already downloaded? Maybe we
should use a icon that has a yellow star (or something that means "new")
on it, but no "file" icon, so people can understand better that the
"file" is not on their computer, but that the item is "new"?

> (c) syncing in the GUI did not work: the whole gpodder interface just 
> froze up. But the command line gpodder --sync worked well.

Which iPod are you using? Can you please post the console output when
running gpodder with the "--verbose" switch, i.e. open a terminal and
enter:

gpodder --verbose 2>&1 | tee ~/Desktop/gpodder-log.txt

Try to reproduce the bug and then post the "gpodder-log.txt" file that
should appear on your Desktop. Before posting, make you you check the
logfile for any sensitive information (if any) and remove/anonymize it
from the logfile, if you think it's irrelevant for the bug.
