Hi there!

After the post from yesterday, I've been preparing an installer for 
PyGTK (based on Alberto Ruiz' PyGTK All-in-One installer, but with newer 
  versions of all packages and some DLLs added to fix some issues with 
older versions and after that also a nice installer for gPodder 0.13.0 
(actually, it includes all 0.13.1 fixes, but 0.13.1 is not released yet).

You have to install PyGTK first (the installer from that link installs 
Python 2.5 + GTK + PyGTK) and after that install gPodder. It should 
work, some icons are missing and some things are still too much unix-y:

http://khan.thpinfo.com/~thp/tmp/gpodder-win32/

Instead of using the PyGTK installer from that page, you can also 
install Python2.5, GTK runtime environment, PyGTK, PyCairo and PyGObject 
seperately and then simply install gPodder - it will use the default 
Python installation on your Windows system.

Please give some feedback if it works on your system. If it does work, 
we might be publishing it on the gPodder website as experimental port.


Thanks,
Thomas
