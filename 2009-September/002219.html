Hello!

gPodder needs your help! :)

As part of the refactoring, I'm splitting more of the
UI between the Desktop and Maemo version of gPodder.

This allows us to create well-designed UIs for each
environment while sharing the same underlying logic
for things like download management and playback.

The current shownotes dialog looks a bit boring and
misses some features that can only be reached from the
context menu of the episode list in the main window.

I'd be very thankful for suggestions, mockups or even
GtkBuilder .ui files so I can combine all suggestions
to create a really usable and useful shownotes dialog
for the next version of gPodder.

Minimum UI requirements:

 * Play / Stream
 * Delete
 * Download
 * Pause download
 * Resume download
 * Cancel download
 * Visit website
 * Progressbar / status area for download progress
 * Text area / other UI for displaying the shownotes

Please also read this bug with all the comments and
duplicate bugs that have some feature requests:

  https://bugs.gpodder.org/show_bug.cgi?id=461


Thanks,
Thomas
