On Sun, Sep 20, 2009 at 5:03 AM, Bernd <brot at gmx.info> wrote:
> Hello Thomas,
>
> what's the reason why you don't use notify-osd on Ubuntu baseed systems?
> I don't like the custom notification windows in gPodder on Ubuntu.

There is nothing custom about gpodder's notifications.  gpodder is
using the standard pynotify interface available as part of the python
gtk bindings that wraps the libnotify library which implements the
freedesktop.org notification specifications.  The traditional
notification daemon that comes as part of a standard GNOME desktop
handles all possible allowable notifications..including notifications
with actions.


Notify-osd is specalized notification daemon which only handles a
subset of possible notifications allowed by the notification
specification. Notify-osd does not handle notifications with actions
for example, and turns such notifications into dialog windows.  This
was a deliberate design choice by Canonical and it would be a
misstatement to suggest that Notify-osd's behavior is "standard"
interpretation of the notification specification.   Gpodder would have
to give up the use of actions in notifications in order to work
smoothly with notify-osd's design choices.
