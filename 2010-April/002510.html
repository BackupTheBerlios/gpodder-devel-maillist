Hello!

As a bugfix release on top of yesterday's 1.3 release, mygpoclient 1.4
fixes some unit tests and adds some more doctests to verify correct
behaviour. This is the version that is now a dependency of gPodder's Git
development version and it will be a dependency of the next gPodder
release.

If you are using the Git version of gPodder, you can install mygpoclient
1.4 from the gPodder Ubuntu PPA. If you don't run Ubuntu, please ask
your packager to package mygpoclient 1.4 for your distribution. You can
also use mygpoclient directly from Git by checking out "mygpoclient.git"
from repo.or.cz and then symlinking the "mygpoclient" Python module into
the "src/" subdirectory of the gPodder checkout.

This change was necessary to support new features of the gpodder.net
webservice such as playback status synchronization. As a nice side
effect, we will also gain the ability to remember the playback position
and track length inside gPodder.

To make use of this feature, you need to be running a recent Git version
of Panucci. I'll create a package of Panucci for the Linux Desktop soon,
so you can use Panucci as your audio player and benefit from the new
features.


Enjoy!
Thomas
