Hi all,

On certain players you can install the rockbox software. This does not wipe out the original firmware. The player really becomes a dual-boot player, using either rockbox or the original firmware. The rockbox software provides exactly the desired properties:

> - It would be nice to somehow have podcasts and music
> seperated on
>   the player. I don't want a "music library" where I

Rockbox presents itself as a "mass storage device", so you can put the files whereever you want to put them. I have a directory "podcasts", underneath of which gPodder stores the podcasts, one directory per channel.

> - The player should remember the position for each single
> podcast
>   and resume there later. I often switch between

Mmmm..... Rockbox definitely does this for the file it is playing when you switch off while playing. I don't know if it can keep track of "stopped position per track".

> - The fast-forwarding shouldn't be too slow. I hate it when

In rockbox, the fast forward (and reverse) speed increases while using, so even a three-hour episode can be FFWD in a matter of seconds. The "FFWD speed up" can be set up to individual wish.

The only downside of rockbox is that only somewhat older models of players are supported. See the website 

http://www.rockbox.org/ 

I use rockbox on an iPod 5th generation (iPod Video, 30 GB), and it works very well. In my case, I would like to get a new player (mine is getting old), but I want an Apple because of all the surrounding infrastructure: docking stations at home, connector in the car, etc. But rockbox cannot be installed on the latest iPods, and iPod + linux does not mix very well :-)) Apple is very clever: by allowing many third parties to make all kinds of accessories, they make sure that once you have an iPod, you will likely buy a new iPod, because otherwise you'll have to buy everything new....
