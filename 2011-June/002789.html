I think it is a good idea to use an external download manager.  Not
only for mobile devices, but in general.  So that gPodder would focus
on podcasting and not deal with bandwidth throttling, etc.

My understanding is that there would need to be an API to queue an URL
for downloading, get asynchronous updates and pause/cancel specific
file, maybe pop up a settings dialog.  The rest would be done by an
external module which would use Woodchuck on Maemo, something else on
desktop, support plugins etc.  gPodder would have a local
implementation based on download.py, in case the external module does
not exist.
