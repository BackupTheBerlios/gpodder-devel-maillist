> Where can I find the to do list?

The current gPodder TODO list can always be found here:

http://svn.berlios.de/svnroot/repos/gpodder/trunk/TODO

> Shall I file a bugreport for my wishes or improvement ideas?

For wishlist items, you can send a mail to the gpodder-devel mailing  
list and I'll add it to the todo list if nobody else is doing it.

For real bugs, it's better to file a bugreport on gPodder's  
berlios.de project page and bug tracker: http://developer.berlios.de/ 
projects/gpodder, but many people find it easier to simply send the  
bug report to the gpodder-devel mailing list, which is also fine (but  
a bug may be overlooked for some time..).

gPodder doesn't only run on Linux, there is at least one active user  
I know of that uses some flavour of BSD. Also, gPodder keeps a local  
database of the downloaded podcasts' metadata, so just removing the  
downloaded files is not enough.

>> Combined with this feature is also a "delete locally-deleted  
>> podcasts on
>> the iPod/MP3 player" feature wishlist item, which would also help in
>> keeping the portable player clean.
> looks as if this could be solved with rsync or unison.

Maybe this would work for filesystem-based MP3 players (but then  
again, gPodder's download directory structure isn't the same as the  
current "synched" format, which basically sorts files into folders  
and easily-readable file names.

And of course this method would not work at all for iPods, as we have  
to update the iTunesDB, etc..
