gPodder 0.9.1 is out since early April, you have to get the newer
version which includes a fix for your problem.

Sadly, no one of the Gentoo package maintainers has updated a new ebuild
to Gentoo, so you still cannot go the official Gentoo way (can somebody
please file a bug in the Gentoo BTS so that the new version gets
packaged?).

You can easily procede to the 0.9.1 release by downloading the source
file from the gPodder website or even go to the bleeding edge subversion
trunk head, instructions can also be found on the gPodder website. You
can then run gPodder from your checked out svn working copy by using
"make test" or creating a shortcut to "bin/gpodder --local".
