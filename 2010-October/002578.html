Dear gPodder developers and users!

The gpodder.net Client Library, mygpoclient, has been updated, and
version 1.5 is now available for download. This is a bugfix release and
does not yet contain any new features that are exposed through the API.

The following bugs are addressed by this release:

  * Limit the number of authentication retries
    When a wrong username/password combination is passed to the library,
    mygpoclient will try to authenticate to the service forever. This
    fix limits the maximum number of retries to 3, failing after the
    third unsuccessful attempt.

  * Accept unicode strings (gPodder bug 1131)
    In certain setups, gPodder was too strict with the checking of
    incoming data from the web service, and incorrectly flagged unicode
    strings as invalid URLs, which prevented clients from retrieving
    subscription list updates from gpodder.net.
    Thanks to Goetz Waschk for reporting this issue and testing patches.


This update is recommended for all users, but not mandatory. You can
grab the latest source code from the mygpoclient website:

  http://thpinfo.com/2010/mygpoclient/


Enjoy!
Thomas Perl
