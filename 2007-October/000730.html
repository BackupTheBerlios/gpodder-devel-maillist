Hi all,

I think there is already a reasonable alternative for
this, by using the 'copy to file' option of gPodder.
Double-click on an episode, and a dialog with info
comes up. You can choose to save a copy of the episode
to any location on disk, and the copy will have the
same name as would appear on an FS based player
(although this option still needs some development, it
seems to do what the original posters wants it to do,
i.e. save a copy for use in another player).

I agree with Thomas about the need to come up with an
absolutely fool-proof way to determine a file name. It
is good programming practice to make something that is
*guaranteed* to work, instead of something which *may
not* work under certain circumstances. As a developer
one really cannot expect to write a contingency code
for each perturbation of errors that may occur in an
input.

Consider as an example the Monte-Carlo software MCNP.
This package has been under development for several
decades by some really clever people. When it reads
its input file, some 1300 checks are performed to
*make sure* that the user is requesting a valid
calculation. Because the code has many options, some
of which are mutually exclusive, some of which
influencing each other etc, this check is necessary.
Inspite of making 1300 checks, it is still possible to
write input that will result in a crash during
execution, to the annoyance of the user. Moral of this
story: sometimes it is better to prefer a bullet proof
solution over something that is (perceived to be)
easier to use.

Wilfred
