Thanks for your reply Thomas. I should have read more into the mailing 
list archives.

I can see why you chose hashes and can also see I'm a bit naive. Things 
always look simple until you have to do them yourself. From your 
description I can see that gpodder needs guaranteed unique directory and 
file names. I presume (but correct me if I'm wrong) that cases when the 
title is missing, contains characters unsuitable for file or directory 
names, is longer than a desired maximum length or is a duplicate are 
outlying cases. That is, the title is only unsuitable quite rarely. If 
this is the case then I would advocate using hashes as a fallback 
system. So if it's possible to use the title according to some set rules 
then the title is used and if not then use a hash. If the set rules are 
always used then the program can always tell if it should use the title 
or a hash.

That's my idea. I can see some other people are interested too. Can you 
see a bad test case for this idea?

Jay
