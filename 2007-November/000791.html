Hello, Paul!

On Tue, 2007-11-13 at 07:33 -0600, Paul Dunbar wrote:
> I had this same problem.  Removing ~/.config/gpodder/feedcache.db and 
> restarting gpodder fixed the issue.

It would be good if you could send me your feedcache.db the next time
this problem appears. (the file contains a dict of the feed URLs you are
subscribed to and the RSS/Atom contents of these URLs from the last
"update" run, so if you have subscribed to some feeds you might rather
not let me know about, don't send me that file)

> I'm wondering if the feedcache.db layout had changed during a recent
> upgrade.  I didn't have this problem until upgrading to 0.10.1 from
> 0.9.5.  Downgrading back to 0.9.5 also resolved the issue.

feedcache.db only came into gPodder with 0.10.0. Before that, feeds were
"cached" in a subdir of ~/.config/gpodder/.

I've fixed some feedcache-related problems in one of the latest commits,
so you might want to look out for the next release of try out the SVN
version of gPodder, it's development code and changes more often, but
most of the time it's fine:

http://gpodder.berlios.de/running-from-svn.html 


Thanks,
Thomas
