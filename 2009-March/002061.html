> Does this mean that every program which uses hal will use MUSIC/Podcast
> as music-folder?

Every program which uses HAL will use whatever that file says, which
currently is either MUSIC or RECORD.  But you can use gpodder.conf to
override that for gPodder.

> If I leave it on MUSIC I have no problem with the sync functionality -
> the podcast don't have to be in a special folder, but for the playlist
> functionality I only would like to see the podcast files. But if I use
> the MUSIC folder gPodder would list all files in the MUSIC folder.
> That's not good and would change the behaviour of the playlist function

That's because HAL known nothing about podcasts, and gPodder currently
uses whatever folder it could find. Later there'll be a new GUI for
device specific settings which will let you change this.  For now you
can do that manually by editing gpodder.conf in your device's root
folder.
