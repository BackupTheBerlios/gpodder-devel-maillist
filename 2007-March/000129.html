I found out that if an episode title contains the character " the 
filesystem-based synchronization won't work.

When gPodder tries to sync the episode with that character in it, the 
synchronization freezes. I think that's because a filename cannot 
contain the " character. I only tested the " character, but I'm pretty 
sure that the characters | \ / < and > also wouldn't work.

The feed I used is: http://www.podshow.com/collection/5782.xml
I'm using the most recent SVN build (0.9.0+svn20070318).
