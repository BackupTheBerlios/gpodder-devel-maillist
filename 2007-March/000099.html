Hello Michel,

On Sun, 11 Mar 2007 17:56:39 -0400 Michel Salim wrote:
> I just started using gPodder, and I'm noticing that the treeAvailable
> TreeView cannot be reordered.
> 
> This is no problem for most of my podcasts, which are ordered
> newest-to-oldest, apart from this one:
> 
> http://www.clfuu.net/podcasts/CLF_Quest.xml

I've had a look at this podcast feed and it indeed has a "wrong"
ordering of the episodes (oldest-to-newest). I've modified the code in
the current svn trunk, which now checks if the channel can be sorted
and sorts if from newest-to-oldest when parsing the RSS file. This
should stop your bug from appearing. Please check if it does, and if
there is any problem, just mail again :)

> Another change I'd like to propose is to make the dialog box for
> downloading all new episodes for a given podcast use a scrolled
> window, to avoid the height going over the screen height. I could
> provide a patch for gpodder.glade if people are interested.

I've added code to omit the output of some episodes (with info on how
moch episodes have been omitted) to gPodder's svn trunk.

If you're currently not using the svn version, have a look at the
download/development pages on the gPodder website to get information on
how to obtain the latest svn source.


Enjoy.
Thomas
