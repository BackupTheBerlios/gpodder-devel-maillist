Hello Preben,

On Fri, 16 Mar 2007 23:56:50 +0100 Preben Randhol wrote:
> I have just started using gpodder to download different radio programs
> that I would like to listen to on my ogg player[TM]. The problem is
> that if I do a sync then all files are transferred. Usually I just
> want to transfer one or two programs at a time not 600Mb worth of
> programs per week. As gpodder is now I don't see how I can do this.

You can exclude certain channels from the synchronization process by
selecting the channel, open the "edit channel" dialog, go to the
"Synchronization" tab and select "Skip this channel when syncing to my
device". This way, gPodder will not sync this channel to your MP3
player.

> So I thought to patch gpodder with and option that one can choose to
> sync selected files. However, as I look in the code I'm wondering if
> there is an easy way to do it. What I mean is that the
> on_sync_to_ipod_activate function does a lot of checks that shouldn't
> be duplicated?

The only method you would want to change is the
"add_episode_from_channel" function in libipodsync.py. There, check if
the episode is one you want to sync and if not, return before the
gPodderSyncMethod.add_episode_from_channel() function call.

I don't know what's the best way to solve this without much user
interaction (e.g. a "Select episodes to sync" dialog or checkboxes in
the gPodder main window or a checkbox in the episode info dialog).


Hope that helps,
Thomas
