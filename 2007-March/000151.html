Hi Thomas,

Sometime in the past two weeks, Makefile was modified to call
gtk-update-icon-cache within the install target. This might cause
problems with packaging, for both RPM and Deb, since the build is
normally done as a normal user into a separate buildroot, and the
files are only installed under /usr/share/icons at package install
time.

Could a build target be added that installs without doing this?
Alternatively, check if the current user is root or not (but this
might not work in Debian, where -- I'm not sure about this -- packages
might be built under fakeroot)

Also, /usr/share/datadir/gpodder.glade tries to load
gpodder-48x48.png, which it expects to be in the same directory --
this does not work unless one makes a symlink from one of the icon
files to this.
