Hello Alain,

On Sun, 18 Mar 2007 12:51:56 +0100 Alain TAUCH wrote:
> I just co the latest gpodder, and realized that the python path
> changed (in fact since r257). On FreeBSD, the python path is
> /usr/local/bin/python so gpodder won't work. What was the 
> reason for removing /usr/bin/env use ?

While changing the .deb packaging code for Debian, I've changed this to
go with the preferred interpreter location. The relevant paragraph is
"1.3.2 Interpreter Location" in the Debian Python Policy[1].

[1] http://www.debian.org/doc/packaging-manuals/python-policy/ch-python.html

I've changed the code again to work on FreeBSD (/usr/bin/env python),
as it is not mandatory in Debian's policy to use a interpreter path
of /usr/bin/python.


Enjoy,
Thomas
