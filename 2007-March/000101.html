> Would you reckon it will be a good idea to make gpodder runnable this
> way? It would involve not hardcoding the path to the glade file and
> probably to have a "python setup.py run" target that would add the

$ make
make test            run gpodder in local directory
make cl              make new changelog entry (1)
make release         create source tarball in "dist/"
make install         install gpodder into "/usr/"
make uninstall       uninstall gpodder from "/usr/"
make generators      generate manpage, run tepache and resize logo
make messages        rebuild messages.pot from new source
make clean           remove generated+temp+*.pyc files
make distclean       do a "make clean" + remove "dist/"

(1) Please set environment variable "EMAIL" to your e-mail address


You might want to run "make test" to get the intended behaviour :)
