Hi everyone,

Sorry if this is not the correct place to start this discussion. I'm new
to the dev list and I'm not into Gpodder development. I only want the
devs to take some attention on these UI issues and discuss them.

I don't understand why the Update button is on the bottom. I always
found that strange to some extent. Also, showing the status update by
hiding the button seems even more strange to me.

Why are some controls on the top and another one on the bottom? Yes, the
controls on the top are applied only to the selected items, but there's
already some exceptions, Preferences and Exit.

Also, why ditching the status bar? It's a component found on almost
every piece of software and users will fastly know it's use: show the
status information.

Thinking on these issues I made a mockup on Gimp. My English is not very
good, so you'll probably understand better what I'm going to explain by
looking it.

http://twitpic.com/4rdkmm/full

First change, move the Update button to the top, alongside all other
buttons. Separated from the other controls so it's easily understood
that it's not applied to the selection. Maybe a text change would also
be desirable: «Update All» or just «Update» instead of «Check for
Updates».

Second change, show the status in a Status bar, or if you think status
bars are Evil, make use of some Chromium-like magic statusbar, also
present now on Firefox 4 and proposed for inclusion on Ubuntu 11.04. I
don't know if this can be done easily, though.


Thank you.

Jorge.
