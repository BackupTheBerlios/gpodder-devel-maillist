Hello!

If you are using gPodder SVN, please update to the latest version and if
you want to help out making sure that the proposed better file naming[1]
works for all your feeds, please send me your channels.opml files
(directly to thp at perli.net, not to gPodder-devel). You can (and should)
remove any podcasts from the file that you don't want me to know about,
but for all "normal" podcasts, it would be very helpful to get a list
that I can run automated checks on.

These checks make sure that for every episode download URL, gPodder
finds a unique filename. This is needed, because gPodder wouldn't know
which of two episodes you have downloaded if they resolve to the same
file name on your disk.

If you are using the latest SVN version, you can use the menu item
"Subscriptions" -> "Send subscription list via e-mail"  (a new
feature ;) to send me the channels.opml file. If not, you can manually
send the file ~/.config/gpodder/channels.opml.

I'll have a patch available soon for you do to more testing. The result
of this will be that newly-downloaded episodes will have a "nicer" name
in the download folders, making it easier for you to find and identify
downloaded episodes with a file manager.

Thanks,
Thomas
