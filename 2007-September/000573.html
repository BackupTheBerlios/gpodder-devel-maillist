>> I realized that another solution to my problem would be to just allow
>> gPodder to save the podcast files with their original filenames when
>> transferring to a player.  The files I'm using have sequential,
>> numerical names by default.  I think a lot of podcasts name their
>> files with this sort of thing in mind (i.e. rails-055-erubycon...).  
>>     
>
> I've just implemented support for this in gPodder-SVN. Get the SVN
> version of gPodder, as described here:
>
> http://gpodder.berlios.de/running-from-svn.html
>
> Then start your SVN version and go to the preferences dialog, to the
> "Player" tab and activate the "Custom filename:" checkbox. In the entry
> box for the format string, type {episode.basename} and it will use the
> name from the URL. You can also click on the "[i]" button to get help
> about how to use format strings in this entry box.
>
> Hope that makes this wishlist item implemented, please report back on
> how it works for you.
>
