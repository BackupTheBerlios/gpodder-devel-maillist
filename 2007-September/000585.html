gPodder uses libgpod (or more exactly, gpod's Python bindings) to access
the iPod. As soon as libgpod supports the checksum generation (from what
I just read on the 'net, it's just a matter of querying a device ID
[using libusb, maybe??] and then generating a checksum of it and writing
it to the iTunesDB).

For now, you should be able to normally use gPodder and then follow the
manual procedure found here: http://www.backdot.com/?p=50

As soon as the libgpod guys implement the checksum generating code in
their library (and distributions ship new libgpod versions, etc..),
gPodder should work with the new iPods just as well as it does with the
older iPods.

You can always ask about when the checksum generating code will be ready
on the gtkpod/libgpod development mailing lists, the URLs and addresses
can be found on this page: http://www.gtkpod.org/contact.html

I'm still happily using my several years old iPod mini, without
problems :)
