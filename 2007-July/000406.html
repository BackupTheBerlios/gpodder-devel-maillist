Hi all!

I need to download episodes from a feed from my university, which is
password-protected. It uses a simple HTTP authentication.

When I tried gPodder (really nice piece of software by the way) it could
read the feed data by prepending the credentials to the URL
(http://user:password@host/feed.rss), however, downloading episodes
fails. My guess is that the connection to download the episodes has to
use the same credentials, but gPodder doesn't cache or try to use them.

Maybe you could add a user and password field to the feed details
dialog, or handle HTTP 401 "authentication required" errors by
displaying a credentials dialog when needed.
