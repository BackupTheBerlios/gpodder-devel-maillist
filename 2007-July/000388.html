After some discussion on the gtkpod-devel mailing list, Todd Zullinger
has given some hints on how to implement a compatible time_released
setting method for both libgpod < 0.5.x and libgpod >= 0.5.x. The
attached patch applies against the latest svn trunk head (patch -p0 <
gpodder-r345_libgpod-0.5.x.patch).

Please test if this patch works for the new libgpod 0.5.x. The code
falls back to the old method of setting time_released, so users of older
versions of libgpod should have no problem using it, too :)
