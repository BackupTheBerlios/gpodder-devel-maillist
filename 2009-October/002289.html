>
> Hello,
>
> We are a team of 5 university students, including Thomas Perl, and will
> start working on a rewrite of my.gpodder.org.
>
> Currently we are defining targets and communicating our plans to the
> course administration. According to our current plans, we will implement
> the following features:
>
> For users
> * subscription management per device
> * synchronised episode states
> * a web-interface for managing your subscriptions
> * synchronizing subscriptions/episodes between devices
> * a public top-list and podcast directory
> * podcast- and possible episode-recommendations
> * a history of added/removed subscriptions
>
> For developers
> * a more advanced API which can also handle episode states
> * a simple API for "dumb" clients to upload whole subscription lists
>
> We will soon announce details about the API as well as public
> documentation and the repo.
>
> Feel free to comment and send suggestions! :)
>
> We'll keep you updated.
