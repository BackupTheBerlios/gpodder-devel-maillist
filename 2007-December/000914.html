Really strange. Is there something like SELinux running that could
prevent gPodder from running correctly? Most of the problems related to
root being able to run the program and the user not are either because
of permission problems or because of some per-user misconfiguration.
Maybe you want to try to create another "normal" user on your system and
see if it works with this one (to see if it's some config problem) or
simply try to set SELinux to the "watch" mode (and not the blocking one)
or simply disable it for the testing period.
