Hello, Pieter!

Pieter De Decker wrote:
> This is kinda weird. The updated nl.po is present in /data/po, but gPodder 
> doesn't actually use the Dutch interface. How come? Does gPodder use the default 
> English interface if the localized version still contains fuzzy strings?

No, but if you are using the SVN version, you have to run "make
messages" to get the binary .mo files for the text .po files, which will
then be loaded by gettext when you use "make test". As the .mo files are
binary and always generated from the .po files, these files are not
included in the SVN version - they are also not included in the source
tarballs, but are compiled when packages are generated or when "make
install" is run.
