Hello, Jeff!

On Mon, 2008-07-07 at 12:23 -0800, Jeff Spaleta wrote:
> On Thu, Jul 3, 2008 at 2:13 AM, Thomas Perl <thp at perli.net> wrote:
> > I think the easiest way is to patch the default value for the
> "opml_url"
> > configuration variable in "src/gpodder/config.py" (it's currently
> set to
> > directory.opml on gpodder.org). This way, no big changes need to be
> > done, and Fedora users will get a list of fedora-specific podcasts
> > listed.
> 
> I was looking for a way to pre-populate
> ~/.config./gpodder/channels.opml with an initial channel definition.
> So any new installs of gpodder on a f10 system would result in the
> Fedora TV feed being as easily found as possible...and easily deleted
> if not wanted.

Maybe we should install a default "channels.opml" file in,
say, /usr/share/gpodder/, which will be used as default if no
channels.opml exists. This can be an empty list in stock gPodder and
distributions can then easily exchange that single file in their
packages with a distribution-specific channel list.

What do you think?

Thomas
