I'm not sure this is related... but import gtk can fail if the display
is not accessible. I've seen this happen several times now in
configure scripts which attempt to import gtk as a check to make sure
that pygtk is available.  The init process for the gtk module load
attempts to access the graphical display and if it can not, falls over
and dies.  I'm not sure if this is the underlying problem, but this is
what I see sometimes when doing python application builds for Fedora
which want to test for the gtk module. gpodder's make test may fail in
the same way.. if done under a chroot for example.
