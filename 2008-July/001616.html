On Mon, Jul 7, 2008 at 12:35 PM, Thomas Perl <thp at perli.net> wrote:
> Maybe we should install a default "channels.opml" file in,
> say, /usr/share/gpodder/, which will be used as default if no
> channels.opml exists. This can be an empty list in stock gPodder and
> distributions can then easily exchange that single file in their
> packages with a distribution-specific channel list.
>
> What do you think?

That would work just fine for my specific needs.  Though I would be
more restrictive about when it was used. Not just when the
channels.opml doesn't exist, but when the .config/gpodder/ directory
doesn't exist.  If a user wants to nuke their channels.opml file
manually to empty it, then it should not revert back to the
pre-defined default.
