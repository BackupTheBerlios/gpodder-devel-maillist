Thanks for your feedback. See below..

You wrote:
> I've spent many hours chasing bugs on 0.14.0 "A Short Drink From
> a Certain Fountain".
> 
> Here's my list of bugs found. I want to hear out what you think
> of them before reporting on the bug tracker.
> 
> 1 - When you go to iPod -> "Remove episodes from device", the
> "Copying files to device window" shows up in addition to the
> "Delete episodes chooser window". Screenshot will be posted if
> needed.
> Correct behavior would be for the "Copying files to device window"
> not to show up.

Please file this in the bug tracker. It's an UI bug.

> 2 - Tooltips not showing up on the episode list. When you hover
> over an episode, a tooltip used to show up with shownotes for the
> specific episode.
> Solution: Right click on episode and select "episode details" option.

This change is deliberate. You can over over the icon of the episode
or podcast to see the tooltip.

> 3 - On some podcasts, "Buzz Out Loud" for example, when right
> clicking on episode and selecting episode details option, gpodder
> freezes for several seconds before showing details.
> Please note that in most podcasts this doesn't happen.

This is because you are probably using the HTML shownotes and the
podcast in question uses embedded images? This has been fixed in
the development version already, as that will have a threaded
downloader for the shownotes that should not freeze.

> 4 - When gpodder is killed by accident instead of gracefuly
> closed. Podcast list is corrupted and all episodes downloaded
> before killing it don't show on gpodder's list even though they
> take up hard drive space.
> Quick solution: Delete all folders on the gpodder download
> folder then re-download your episodes.
> This can probably be avoided by having gpodder save it's database
> after downloads are finished.

Yep, please file a bug about this in the bug tracker. We do not
store the database changes during runtime now to have a better
UI performance on low-end devices. We should save the database
every now and then, though.

> I really liked the new features added to gpodder.
> My favorite was my.gpodder.org

:)

Thanks for taking the time to review it and prepare this list 
of potential bugs.
