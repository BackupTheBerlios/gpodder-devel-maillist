Wget will "put" the file in the current directory. Eg. if you are in ~ 
(/home/your-username), that's where the file will be placed. You can find the 
current directory using the "pwd" command.

Generally you would download the source tarball to somewhere in the home 
directory of an unprivileged user (like ~/sources/ for example) and install it 
as root using "sudo make install" or "su -c 'make install'". Once it's 
installed it will be available to all users on the system.

> On another note, I did a "make install" the first time and I got a python

You're probably missing some tools to build and install the package. Since 
gPodder is already in the Fedora repos you can install them automatically 
using yum-builddep. Here's a more detailed description on how to install 
gPodder on Fedora from source.

# install yum-utils to get the yum-builddep program
su -c "yum install yum-utils"

# fetch the build dependencies for gPodder
su -c "yum-builddep gpodder"

# make yourself a directory to keep your source packages
mkdir ~/sources

# change to that directory (same as doing cd /home/$USER/sources/)
cd ~/sources

# download the tarball to the current directory
wget http://download.berlios.de/gpodder/gpodder-0.16.1.tar.gz

# extract the tarball to the current directory
tar xvf gpodder-0.16.1.tar.gz

# change to the gpodder source directory
cd gpodder-0.16.1/

# build and install the package in one step
su -c "make install"

And that should be it, gPodder should now be available from your applications 
menu and you'll be able to launch it from the command line using the "gpodder" 
command.
