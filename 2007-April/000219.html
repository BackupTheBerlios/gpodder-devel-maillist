I've tried to download the most recent episode (quite an adventure on
64k ;) and found gPodder react the way you reported. I've fixed this bug
by filtering non-enclosure items earlier in the parsing process, so
gPodder will now never "see" non-enclosure items (as opposed to omitting
it in the display, which was the default behaviour until now). Check out
the latest svn revision with this bug fixed :)
