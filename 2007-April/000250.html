It's me again. I found out that I can also use the experimental Debian 
packages. In other words: they're not specifically made for Ubuntu. libgpod 
0.3.2 (aka libgpod0) is still on my system, but I've also installed libgpod 
0.4.2 (aka libgpod1) and that worked for me.

You might want to use this for an FAQ or something: 
http://ftp.debian.org/debian/pool/main/libg/libgpod/python-gpod_0.4.2-1_i386.deb
http://ftp.debian.org/debian/pool/main/libg/libgpod/libgpod-common_0.4.2-1_all.deb
http://ftp.debian.org/debian/pool/main/libg/libgpod/libgpod1_0.4.2-1_i386.deb

When trying to install python-gpod 0.4.2, dpkg started complaining about 
python-central not being recent enough. I resolved this dependency problem by 
installing python-central 0.5.13 from Debian's FTP server:
http://ftp.debian.org/debian/pool/main/p/python-central/python-central_0.5.13-0.1_all.deb

Please note that these packages are EXPERIMENTAL. Another thing you might want 
to know: I'm a user of Kubuntu 6.10 but I'm on the GNOME desktop.
