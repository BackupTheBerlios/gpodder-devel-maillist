Thanks to Nick, we now have basic BitTorrent support in gPodder :) If
you want to use the default method, you will need gnome-btdownload.

Also (as suggested by Paul Elliot last week) we now have support for
download bandwidth throttling on a per-episode basis. You can enable the
bandwidth throttling in the preferences dialog.

If anybody wants to help with the download queue (e.g. "download max. 3
episodes simultaneously), feel free to send in patches :) We already
have some code for that in place in src/gpodder/console.py, but the
download queue should work with threading and GTK and also display
what's downloaded and what's queued in the download status tab.
