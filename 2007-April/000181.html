> lower than 0.9.1. This makes is possible to upgrade from 0.9.0 to all

Ah, that makes perfect sense. The closest within the Fedora versioning
guideline would be '0.9.0.${date}', but AFAIR the pre-release tag is
supposed to be encoded in the release field.

A bit academic, in any case, since the last stable release is more
than usable for most. And Fedora's naming of pre-release kernels is
also based on the last stable version (even though the API has
changed), so gPodder is in good company :)
