Hello, Nick!

On Wed, 2007-04-18 at 22:55 -0400, nikosapi wrote:
> Ok, I've just spent my evening having some fun with gPodder :-)
> I added your patch and fixed some libipodsync bugs.

Thanks, I can't test video, so I'm glad you added these fixes. Hopefully
this makes video podcasts better supported on iPods..

> If you now go into the gPodder sync properties tab there is an option
> to sync only unplayed episodes (which was a great idea, I'm now using
> it!). I had to add a bit to your patch so that you can still add
> episodes to the device while "only sync unplayed ..." is on. This can
> be achieved by selecting the episode you want to add and clicking the
> transfer button. The played episodes aren't synced only when you do a
> full synchronization (CTRL + s). 

Sounds good to me.

> The set_podcast_flags function in libipodsync it would overwrite all
> the mediatype settings to audio 0x00000004, so when you would sync
> again after adding a video it would no longer show up as a video. I
> just moved that code to where the video detection takes place.
> track.mediatype doesn't need to be reset each time a sync takes place.

Agreed.

> Also, I set the videos filetype to m4v instead of everything as mp3,
> it seems to have fixed a random bug where some of my videos would be
> still regarded as audio (but this might not be necessary, my ipod
> could just be crazy).

Do .mov files (if they even work on an iPod?) also work when this is set
to m4v?

> Lastly, I made it so that torrent files couldn't be transferred to
> devices, I don't see why you'd need torrent files on a media device. 

I've changed that part so that it doesn't check for "isn't torrent", but
instead checks for "is audio or video". That should solve problems with
feeds that provide other kinds of files (I think I've seen a podcast
feed with a PDF transcript recently..).

Please apply the attached patch to your working copy (r313) and tell me
if it works as expected. I'll commit it to svn then.


Thanks for your work,
Thomas
