On Wednesday 11 April 2007 20:00:33 Darren O'Neil wrote:
> Okay, I have libgpod0 version 0.3.2-1.1ubuntu1 installed. I do not have
> libgpod-dev installed. python-gpod installed is version 0.3.2-1.1ubuntu1
> as well. gpodder is 0.9.1-1

The problem is that libgpod is too old, the latest version is 0.4.2 .
I would recommend uninstalling all of your libgpod related packages and 
install the new version by following the instructions found on this page:
http://gpodder.berlios.de/dependencies.html If you have any problems 
whatsoever doing this feel free to ask and we'll give you a hand.

> Really don't know what top or htop is. Using latest ubuntu distro. It is
> a slower machine I am running on. My internet is not super snappy
> either. Any way I can limit the concurrent downloads to say three or
> four?

Hehe, you "were" using the latest Ubuntu distro. The newest one (to be 
released April 19th) is 7.04 (A.K.A. Feisty Fawn) which is available as a 
beta now and includes libgpod 0.4.2 .
FYI top and htop are process monitors, I'd recommend htop it's a fancier 
version of top. It'll tell you what your computer is doing (how much 
cpu/ram/swap is being used and by what processes).
