Hallo Thomas , 
thank you for taking the time to have this constructive discussion with us.

My opinion to your suggestions:

>    * Only show "New episodes are available" once for a set of episodes
OK

>    * Remove all the buttons on the notification bubbles
OK.
They are there for historical reason. Somebody suggested to have the nice countdown clock in notification. As I remarqued this need at least one action, I added the "show" button. This button whas funny, I started to impulsively add others without real background reflexion on their utility. Your suggestion of a "hurry, hurry - you have to do something now!" effect as conviced me they are more disturbing as helpfull.

>    * Remove the current automatic download option (default behaviour)
OK in it's actual beheviour: open an episode selection dialog for dowloading.

>    * Add option to download new episodes in the background (minimized)
OK (off by default)

>    * Make sure automatic downloads never show the gPodder window
OK

This points are nice for having gPodder running in background. I would be happy with them.
But I think there is still a question open: what to do when gPodder is NOT in background, and an automatic update appends. I thing having popups in this case might also be disturbating. Replace them with notification is not a good solution since it can be deactivated.
