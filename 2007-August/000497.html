Yes, I'm currently reorganizing some parts of gPodder. If you have local
changes or any other differences, "svn update" might not pull in
everything correctly. If you have local changes but don't mind them
being overwritten, you can do the following in your local SVN gPodder:

svn revert -R .
svn update

This should revert any local changes and pull the latest changes from
the repository.

Ubuntu's python-id3 package is not the PyID3 package that gPodder needs
to extract cover art from MP3 files, see:

https://lists.berlios.de/pipermail/gpodder-devel/2007-July/000799.html

If you still want PyID3 support, Nick has created some packages you
could try:

https://lists.berlios.de/pipermail/gpodder-devel/2007-July/000806.html
