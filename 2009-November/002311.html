Hi,

As expected, we got the OK from our course administration today, so
work on my.gpodder.org can finally start! I can now also disclose our
planned features and a rough schedule.

Features
* a simple API (similar to the existing one) for quick implementation
in other clients
* an advanced API that can also sync episode states and send
subscription-diffs states
* podcast-information (descriptions, episodes, etc) on the website
* a private history of subscription and episode state changes
* public toplist
* suggestions based on other users subscriptions
* privacy settings to exclude the whole profile or some subscriptions
from toplist/suggestions
* support for multiple devices (subscriptions per device, global episode-states)
* integration in gPodder, of course ;)

We plan to have a very basic version out within one optimistic week.
Advanced API, gPodder integration and toplist should be finished
before christmas. The project is scheduled to be complete before the
end of the semester, about mid-January.

I'm looking forward to hearing some suggestions and comments from you!


-- Stefan
