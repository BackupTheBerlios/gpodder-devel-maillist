Hello Thomas,

The following patch attempts to address two items on the TODO list.

The suggestion from Robert Young about having no write permission to
his file-system based mp3 folder, causing gPodder to hang, is handled
with a quick check for write permission using the `os' facilities and
creating an error dialog if there is no write permission.

The suggestion from Andy (and recently from Tom Poe) is addressed by
counting podcasts that exist locally and counting podcasts that exist
on the mp3 filesystem when the `Remove podcasts' menu item is
selected.  If there are more on the mp3 filesystem than locally, it
creates a set for both local and mp3 filesystem files and removes the
files that are unique to the mp3 filesystem.  It's not a particularly
elegant solution, but I think that it provides the intended behavior.
